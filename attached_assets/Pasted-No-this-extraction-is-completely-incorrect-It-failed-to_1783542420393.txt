The Diagnosis: What Went Wrong
The Header Offset: The SheetJS library (XLSX.utils.sheet_to_json) blindly assumes the very first row of an Excel file contains the column headers. Because the Computershare file has a title row (COMPUTERSHARE (PTY) LTD) sitting at the top, the script made that the first header and filled the rest with __EMPTY, __EMPTY_1, etc.

Blind Regex: Because the actual headers (NETT CASH, JSE CODE) got pushed down into a standard data row, your regex could not find them. That is why Net Cash is R 0,00 and Unmatched is 4 (it failed to map everything).

Swallowing Garbage Data: Because it is just reading top-to-bottom, the script ingested the TOTAL row and the COMPUTERSHARE CONFIDENTIAL footer row, treating them as if they were actual MINT clients waiting for a payout.

The Fix
You need to update api/dividends/extract.js to dynamically hunt down the true header row before it converts the sheet to JSON, and you need to tell it to explicitly ignore the footer garbage.
